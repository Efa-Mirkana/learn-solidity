# Course Materials Guide

This directory acts as the theoretical companion to the codebase, providing deep conceptual context for each module.

## Notes/
Detailed study guides for every lesson. These files explain the "Why" behind the "How", covering:
- **Security Pitfalls**: Common vulnerabilities associated with each topic.
- **Gas Optimization**: Best practices for writing efficient code.
- **Best Practices**: Industry standards for contract architecture.

## Presentations/
High-level architectural summaries. Use these to understand the "Big Picture" before diving into the code:
- **System Workflows**: Step-by-step diagrams of how Escrow and Voting contracts interact.
- **Inheritance Trees**: Visualizing how contracts relate to one another.
- **Memory vs Storage**: Visual guides to data persistence.

## How to use these materials
1. Read the **Presentation** for a topic to understand the goal.
2. Study the **Code** in the `src/` directory.
3. Review the **Notes** to solidify your understanding and learn about edge cases.
